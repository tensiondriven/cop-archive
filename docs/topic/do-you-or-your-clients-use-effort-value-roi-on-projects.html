Do you or your clients use Effort/Value/ROI on projects?

In Glassfrog, users can currently assign an Effort and a Value to a Project, which will calculate an ROI.  This seems to me like a seldom used feature, and I'm wondering if anyone would miss it were it removed.  (Removing it would simplify the product, which is one of my design goals for Glassfrog. This will be especially important as we continue to build out the "personal system" features in Glassfrog.)

I'm looking to gather some feedback; Do you or your clients use this feature?  If you use it, can you make a case for keeping it?
